# Sprint 0 Acceptance Criteria

Sprint 0 passes only when:
1. Standalone boundary is explicit and testable.
2. No dependency on the existing TradingOS main system exists.
3. Module ownership is non-overlapping.
4. MVP and non-goals are fixed.
5. Public tool boundaries and response envelope are defined.
6. Raw evidence append-only policy and directory contract are defined.
7. Manual execution is fixed for MVP.
8. QA/QC/Verify/Validate policy is documented.
9. Sprint 1 task pack has inputs, outputs, tests, and non-goals.
10. Core documents have version, status, owner, and date metadata.
11. Five core JSON Schemas are valid Draft 2020-12 schemas.
12. Valid examples pass and invalid examples fail.
13. Contract tests verify no scenario probability and no execution permission from sensors/entries.
14. Architecture review authorizes Sprint 1 only.
