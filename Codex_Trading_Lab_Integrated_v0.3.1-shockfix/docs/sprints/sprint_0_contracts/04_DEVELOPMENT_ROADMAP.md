# Development Roadmap

## Sprint 0 — Architecture Freeze — COMPLETE
Architecture, boundaries, schemas, public tool contract, evidence policy, Sprint 1 test matrix.

## Sprint 1 — Fresh MT5 Snapshot
M5/M15/H1, optional H4 context, broker/capture time, closed bars, freshness, account/position context, twenty consecutive real snapshots.

## Sprint 2 — Basic Eyes
Candle features, swing points, basic structure, trend/range/transition, volatility/shock.

## Sprint 3 — Entry-Relevant Eyes
Support/resistance, supply/demand, liquidity map, sweep, break, reclaim, retest, displacement, rejection.

## Sprint 4 — Fusion + Compact Packet
Single public analysis entry point, conflicts, unknowns, evidence IDs, diff output.

## Sprint 5 — Scenario Engine
Primary, secondary, failure, tail-risk paths with required events, invalidation, expiry, rolling updates.

## Sprint 6 — Entry Intelligence
Structured limit, early confirmation, full confirmation, continuation. Measure sharpness and throughput together.

## Sprint 7 — Replay & Training Lab
Blind replay, random drills, session/date/topic selection, deterministic scoring, AI review.

## Sprint 8 — Watcher + Codex Skills
Event-driven jobs, compact prompts, live action plans, manual execution.

## Sprint 9 — Learning Plane
Episodes, hypotheses, experiments, promotion ladder, skill update proposals.

## Promotion rule
A sprint may begin only after the prior sprint has a PASS or CONDITIONAL_PASS report with explicit unresolved risks accepted by the human project owner.
