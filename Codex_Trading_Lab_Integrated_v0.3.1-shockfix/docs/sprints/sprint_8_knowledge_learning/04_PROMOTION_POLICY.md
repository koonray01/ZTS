# Promotion Policy v0.1

## Observation → Hypothesis
- repeated evidence
- at least 3 independent episodes by default
- no future leakage or data-integrity violation

## Hypothesis → Research Finding
- registered experiment
- minimum 20 triggered cases
- frozen definition/version
- result may be SUPPORTED, INCONCLUSIVE or REJECTED

## Research Finding → Validated Finding
- minimum 50 validation cases
- locked OOS evidence
- no blocking integrity issue
- no stream mixing
- statistical metrics attached

## Validated Finding → Candidate Rule
- change proposal
- impact analysis
- backward-compatibility assessment
- shadow plan

## Candidate Rule → Approved Policy
- explicit human approval
- tests pass
- shadow validation pass
- new immutable version

## Approved Policy → Skill Update
- dependency check
- explicit approval
- skill manifest version bump
- no automatic deployment
