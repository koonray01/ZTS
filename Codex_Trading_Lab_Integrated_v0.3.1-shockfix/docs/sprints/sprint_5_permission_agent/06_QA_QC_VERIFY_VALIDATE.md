# QA / QC / Verify / Validate

## QA
- gate unit tests
- integration tests
- schema tests
- negative and safety tests
- hash-chain tests

## QC
- one run/snapshot/candidate identity
- policy and dependency versions
- decision expiry
- duplicate request detection
- safe account masking

## Verify
- WAIT cannot create a proposal
- APPROVED cannot auto execute
- deterministic FAIL cannot be overridden
- stale snapshot cannot inherit old approval
- tool gateway rejects unlisted tools

## Validate
Later measure:
- false approval rate
- blocked winners / blocked losers
- approval latency
- opportunity throughput
- expectancy after permission
- token use per Codex job
