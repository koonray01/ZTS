# QA / QC / Verify / Validate

## QC — runtime data quality
- freshness
- synchronized timeframes
- closed bars only
- gaps/duplicates
- indicator version match
- mixed-time rejection

## QA — engineering quality
- unit tests
- integration tests
- golden fixtures
- regression tests
- red-team cases
- schema tests

## Verify — implementation versus specification
Examples:
- wick through a level is not a close-confirmed break
- scenario nodes cannot be skipped
- stale data cannot become live permission
- Codex cannot override deterministic results

## Validate — market usefulness
Measure:
- opportunity rate
- candidate rate
- entry latency
- blocked winners versus blocked losers
- false approvals
- missed valid setups
- expectancy by entry type/regime/session
- token cost per useful decision
